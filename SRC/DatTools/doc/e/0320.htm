Field Content

Select MakeDat, then Field Content to open the field viewer.
At the top of the dialog there are two spinbuttons: The left one selects the current entry, the right one the current subfield. Both spinbuttons remember their content, even if you close the dialog and open it again. If a stored value exceeds the limits of the currently selected field, it is reset to zero automatically.
Numeric data types with a width of 8, 16, 32 or 64 bit are shown in the first entryfield (the box in the middle):
Push zero to reset all bits in the current entry or max to set all bits. The latter one sets bytes to FF, words to FFFF, double words to FFFFFFFF and quadruple words to FFFFFFFFFFFFFFFF. To enter a specific value, overwrite the content of the entryfield, then click on any other control to evaluate the changed content. Numbers generally are evaluated right to left. Surplus digits beyond the valid range of the current data type are ignored silently.
Data types DO, FIX and $32 are shown in the MLE at the bottom. Strings (type $32) are displayed as plain text:
All other types are displayed as hexdump:
Of course, the MLE's content is editable. It is strongly recommended to keep the format of the hexdump as is - the evaluation function assumes data are at the same places where the output function stored them.
Save writes the changed content back to the field before the dialog is dismissed. Discard resets the changed flag before the field is freed - all changes are lost if you push this button!
