MakeDat

MakeDat in the main menu provides tools to create and manage data fields:
  • Create File
  • Create Field
  • Field Content
  • Move Entries
  • Copy Field Content

Create File

Select Create File. The file creation dialog is opened:
This tool creates a file filled with zeroes:
1. Enter the path where the new file shall be stored.
2. Enter a name for the new file.
3. Enter the wanted size of the new file. The entered size is rounded up to the next multiple of 512 (sector size). The tool denies to create empty files (size zero).
4. Push Create to create a file with the specified properties.
5. If no error report pops up, the new file is available for access.
You can load this file with the Loader, then use the returned MemHandle to fill the file with content, resize it or free the allocated memory block. Because the Loader only supports saving of data fields at the moment, your program has to write the content of the allocated memory back before the memory block is freed. Please do not call database engine functions with the MemHandle of any memory block. Memory blocks have no valid DBE header and are rejected by FDacc(), FDvpt(), SftUp() and SftDn().

Move Entries

Moves entries in the currently selected working field up or down. Choose the first entry to move (source) with the first and its destination (target) with the second spinbutton. If source is larger than target, entries are moved downwards. If source is smaller than target, entries are moved upwards. If both are equal, you trigger an error message. Choose the amount of entries to move. It must fit into the field, otherwise you trigger another error message. Push Move to move entries or Abort to close the dialog without moving anything.
Caution: Entries in all subfields of the working field are moved!

Copy Field Content

Copies the content of the selected source to the selected target. Select source and target by pushing the corresponding Select button. Push Copy to start copying or Abort to close the dialog. Entries are not copied if the defined data type for a subfield in source differs from the data type defined for the same subfield in target. If source holds more entries than target, only those entries fitting into target are copied.
