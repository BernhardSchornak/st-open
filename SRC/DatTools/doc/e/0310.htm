Create Field

Please notice:
1. DatTools assume that all numbers are entered in hexadecimal notation.
2. The content of each entryfield is evaluated whenever it loses the input focus. After you have entered a parameter, click on another control to signal DatTools to evaluate it. This is done because the output function formats the content of the entryfield. Moreover, the cursor position is set to the rightmost character because it is written as last character. If all changes were evaluated, your input were reformatted with every change and you had to set the cursor back to the digit you want to enter.
Select MakeDat in the main menu, then Create Field. The field creation dialog pops up:
Most buttons are disabled for now. They are enabled step by step when you enter required parameters. First enter a field number of your choice. Push Search to tell DatTools to search for the next free field number and set the entryfield to it:
As shown in the screenshot, DatTools automatically assign a string with the field number in hexadecimal notation as the field's name. You should change this to a unique name with up to 27 characters. It is easier to find a field called "spinbuttons" in the field selection dialog than to remember its field number "FFFFFF01":
Now it is time to enter the required amount of entries in your field. This parameter is required for all data types - you cannot create fields without this information!
Whatever you enter as amount is rounded up to the next multiple of 16, so it's a good idea to set the amount to a number ending with a zero. Numeric data types with a size of up to 16 byte (DB, DW, DD, DQ and DO) have predefined sizes, so their corresponding buttons are enabled after the amount of entries is known. If your field does not contain fixed length data blocks or dynamic strings, you can skip the next step and enter required data types.
If your fiels contains fixed length data blocks and/or dynamic strings, DatTools need to know their maximum size:
Please notice: The entered size is valid for all subfields in this field. If you need blocks with different sizes, you have to create one data field for each size. It is impossible to have more than one size in any data field. The size for dynamic strings is the maximum size for any string. if a string passed to FDacc() exceeds this size, the string is truncated before it is written to the field's end. Like the amount of entries, the entered size is rounded up to the next multiple of 16 to allow fast copying via XMM registers.
Finally, you should add the data types required for your new field:
The entryfield min. shows the initial field size after creation, max. the maximum field size if the field is filled with dynamic strings of maximum size. If your field does not contain dynamic strings, min. and max. must be equal.
When DatTools got all required information, the Create button is enabled. After adding all data types, push this button to create the new field:
When the field was created successfully, the View button is enabled. Push it to view the Field Content or Close to dismiss the dialog. The field is available now:
