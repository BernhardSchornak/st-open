Installation

Installation Procedure

To install DatTools, drop "md5.zip" in a folder of your choice and UNZIP it. The DatTools folder is created:
You cannot start DatTools without GCC/2's C runtime library (something similar to "emx.dll"). Please move or copy "dllc101.dll" to a folder like BOOTDRIVE:\OS2\DLL or any other folder in the LIBPATH statement found in your CONFIG.SYS:

If you are using other ST-Open programs, this DLL should be at the right place - there is no need to copy it more than once.
Sidenote: DLLC101.DLL only is required to compile the program properly. The only function ever called from this DLL is __main. DatTools themselves only call Intelligent Design funktions provided by ST-Open's libraries. These functions are integral part of the program's code to guarantee the full utilisation of your processor's capabilities and performance.

Starting DatTools

Just doubleclick on the program object, wherever it may roam. The default place is the BIN or SOURCE folder, but any other folder - except DATA and SYS - within the folder DatTools is perfectly okay. DatTools silently ignore attempts to drag & drop things on them. Passing command line options to DatTools is like pouring some water on a stone to watch it grow.

About Desktop Objects...

It is the philosophy of ST-Open to leave the user's system untouched. This includes not to create objects on the user's desktop, to add no bloat to the user's INI files nor to do anything the user is not aware of. If you love a desktop filled with objects, you are free to drop a reference to DatTools or the program object onto your desktop. I prefer to have a clean desktop without any objects hanging around and I hate to remove those objects when I install something for a short test. Hence, I avoid these - IMHO - bad manners in my own programs.

Things You Should Not Do

ST-Open programs are based on ST-Open libraries. The most utilised function is ST-Open's database engine. Program flags, parameters, strings and other data are stored in files called data fields. These fields are stored in the DATA and SYST folder. Neither move any field out of its folder nor delete or modify any field. It is very likely the program will not run or deny to work if data required to perform a specific task are missing.
