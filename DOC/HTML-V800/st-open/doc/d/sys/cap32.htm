AloMem
Sandkasten
rc = AloMem(groe);
Aufruf
RCX
groe
Grösse des zu allozierenden Speichers
Eingabe
RAX
rc
Adresse oder NULL
Ausgabe
AloMem() ruft VirtualAlloc() auf, und stellt anschliessend die von der API überschriebenen Register ECX, EDX, R08, R09, R10, R11, XMM4 und XMM5 wieder her.
Verwenden Sie Sandkästen nicht in Hochsprachen. Mit schlampigen Konventionen arbeitende Kompilierer und API-Funktionen überschreiben die mühsam wiederhergestellten Register sofort wieder mit ihrem gerade anfallenden temporären Datenmüll. Nur saubere Programmierung profitiert von Registern, deren Inhalt auch über Funktionsaufrufe hinweg unverändert bleibt.
Beschreibung
{
 ...
 ea = AloMem(0x00010000);
 ...
 ...
 rc = FreMem(ea);
 ...
}
Alloziert einen 65.536 Byte grossen Speicherblock. Die Variable ea enthält bei der Rückkehr entweder die Adresse des zugewiesenen Speicherblocks oder Null. Bei Rückgabe von Null kann der Fehler per GetErr() abgefragt werden. Der Speicherblock sollte nach der Verwendung mit FreMem() wieder frei gegeben werden, um den Speicherverbrauch des Systems so gering wie möglich zu halten.
