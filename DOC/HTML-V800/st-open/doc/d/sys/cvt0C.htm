Ausgabe
ph2s() wandelt [paras] Paragraphen (16 Byte) ab Adresse [quelle] in eine formatierte ASCII-Zeichenkette ab Adresse [ziel] um. Nach jeweils [format] Bytes wird ein Leerzeichen zur Formatierung eingefügt. Der Zielspeicher muss grösser als 49 * [paras] plus zwei pro 16 Paragraphen (eine Seite) sein!
Mit dieser Funktion lüsst sich sehr einfach ein (sehr simpel gestrickter...) Hex-Editor realisieren. Der Feldeditor der DatTools verwendet diese Funktion zur Darstellung von Datenblöcken.
Beschreibung
{
 DB s[32] = {
             0x01, 0x23, 0x45, 0x67,
             0x89, 0xAB, 0xCD, 0xEF,
             0x00, 0x00, 0x00, 0x00,
             0x00, 0x00, 0x00, 0x00,
             0x00, 0x00, 0x00, 0x00,
             0x00, 0x00, 0x00, 0x00,
             0x00, 0x00, 0x00, 0x00,
             0x00, 0x00, 0x00, 0x00
            };
 DB t[256];
 ph2s(s, t, 2);
}
Die Ausgabe sieht wie folgt aus: 
01 23 45 67 89 AB CD EF 00 00 00 00 00 00 00 00
00 00 00 00 00 00 00 00 00 00 00 00 00 00 00 00
