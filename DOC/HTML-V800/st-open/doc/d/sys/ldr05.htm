LDchk
Funktion
rc = LDchk(CHK_REMOVE);
Aufruf
DD
CHK_REMOVE
Maximale Dauer der Inaktivität im OS/2 HMSh-Format
Eingabe
DD
rc
immer Null
Ausgabe
LDchk() prüft periodisch alle Speicherblöcke auf Inaktivität. Die Zeit zwischen zwei Überprüfungen wird mit CHK_DELAY (im OS/2 HMSh-Format) festgelegt. Wurde auf einen Speicherblock länger als die in CHK_REMOVE (im OS/2 HMSh-Format) festgelegte Zeit nicht zugegriffen, dann wird der Speicherblock freigegeben. Automatisch freigegebene Datenfelder werden vor der Freigabe gespeichert, wenn ihr Inhalt verändert wurde. Die automatische Freigabe kann mit dem Laderkontrollfähnchen LDR_STATIC für jeden Speicherblock individuell abgeschaltet werden. Da nur die Datenbankmaschine das "geändert"-Fähnchen nach jeder Schreiboperation automatisch setzt, ist es ratsam, bei der Anforderung von Speicherobjekten und Dateien das Laderkontrollfähnchen LDR_STATIC prinzipiell zu setzen, wenn der Speicherblock voraussichtlich länger als die in CHK_REMOVE festgelegte Zeit benötigt wird.
Beschreibung
Die dynamische Speicherverwaltung ist optional. Um sie einzuschalten, müssen die beiden Variablen CHK_DELAY und CHK_REMOVE auf Werte ungleich Null gesetzt und beim Programmstart ein Zeitgeber eingerichtet werden. Für den Zeitgeber muss eine WM_TIMER-Prozedur existieren, in der LDchk() dann periodisch aufgerufen wird. Im V700-Skelett ist der entsprechende Code bereits enthalten. Schauen Sie in V700.c und hvar.h nach, wie die dynamische Speicherverwaltung konkret programmiert wird.
