Ausgabe
GetMsg() ruft GetMessage() auf, und stellt anschliessend die von der API überschriebenen Register ECX, EDX, R08, R09, R10, R11, XMM4 und XMM5 wieder her.
Verwenden Sie Sandkästen nicht in Hochsprachen. Mit schlampigen Konventionen arbeitende Kompilierer und API-Funktionen überschreiben die mühsam wiederhergestellten Register sofort wieder mit ihrem gerade anfallenden temporären Datenmüll. Nur saubere Programmierung profitiert von Registern, deren Inhalt auch über Funktionsaufrufe hinweg unverändert bleibt.
Beschreibung
   Nachrichtenschleife (Standard):
   ...
 0:leaq  0x80(%rsp), %rcx   # RCX = EA MSG
   xorl  %edx,       %edx   # RDX = alle Fenster
   xorq  %r8,        %r8    # R08 = kein Filter
   xorq  %r9,        %r9    # R09 = kein Filter
   call  _GetMsg
   testl %eax,       %eax   # WM_QUIT?
   je    1f
   incl  %eax               # gegebenenfalls Sprung
   je    0b                 # zur Fehlerbehandlung
   call  _TrnsMsg           # Nachricht uebersetzen
   call  _DispMsg           #           verteilen
   jmp   0b
                            # Programm beenden
   ...
Holt die nächste Nachricht aus der Nachrichtenwarteschleife ab. Die Nachricht wird in RAX zurückgegeben. Enthält RAX FFFFFFFFFFFFFFFF, ist ein Fehler aufgetreten. Alle anderen Werte entsprechen gültigen Nachrichten, wobei ein Wert von Null WM_QUIT (Applikation schliessen) signalisiert.
