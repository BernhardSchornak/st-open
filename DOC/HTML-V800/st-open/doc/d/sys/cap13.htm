Ausgabe
WnQTx() ruft WinQueryWindowText() auf und stellt anschliessend den Inhalt der vom API zerstörten Register ECX und EDX wieder her.
Hinweis: Die Grösse des Textpuffers ist auf 65.535 Zeichen beschränkt (diese Beschränkung ist durch das Betriebssystem vorgegeben).
Achtung: Anders, als im Toolkit beschrieben, liefert WinQueryWindowText() nicht ein ULONG mit der Grösse der gelesenen Zeichenkette, sondern die Adresse des übergebenen Puffers zurück.
Rufen Sie diesen Sandkasten nicht von C aus auf. Schlampige Programmierumgebungen profitieren nicht von wiederhergestellten Registern. Sie verschwenden etliche Taktzyklen für das unnötige Wiederherstellen von Registerinhalten, die sofort wieder mit aus dem Speicher geladenen Parametern überschrieben werden.
Beschreibung
{
 VOID *p;
 DB    s[32];
 p = WnQTx(hwnd, 32, s);
}
Kopiert bis zu 31 Bytes des Fenstertextes aus dem Fenster mit dem Henkel hwnd in den bereitgestellten Puffer (Byte 32 für die terminierende Null). Die Funktion kann auch sehr grosse Texte, z.B. den Inhalt eines MLE, auslesen.
