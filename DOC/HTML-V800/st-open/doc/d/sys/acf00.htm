Ausgabe
FDacc() ist der Funktionsverteiler von ST-Open's Datenbankmaschine. Mit der Funktion wählen sie aus, welche Operation die Datenbank durchführen soll. Bitte lesen Sie das Tutorial Über Felder - der DataManager (DM), um einen Überblick über die Verwendung der DBE zu erhalten.
Fehler werden prinzipiell in der globalen Variablen FIA_ERROR (0x1FCC[BNR]) übergeben. Der Rückgabewert rc wird auf Null gesetzt, wenn ein Fehler aufgetreten ist. Die einzige Ausnahme ist FDA_ADDRESS, das aus naheliegenden Gründen die Adresse des Feldes zurückgibt. Das erste Doppelwort im Feld ist Null, was für Bytes, Worte, Doppelworte und Zeichenketten innerhalb des zulässigen Bereichs liegt. Bei Datentypen mit einer Breite von mehr als 32 Bit ist der Feldname Bestandteil des Vier- oder Achtfachwortes, respektive des Blocks, die Adresse weist also auf ungültige Daten. FIA_ERROR ist bei diesen Datentypen auf Null zu prüfen, um die Gültigkeit der Adresse sicherzustellen.
Beschreibung
{
 DD rc;
 rc = FDacc(*MEIN_FELD, 0, 0, FDA_READ, 0, 0);
 FDacc(*MEIN_FELD, 1, 0, FDA_WRITE, 0, rc);
 FDacc(*MEIN_FELD, 2, 0, FDA_INC, 0, 0);
}
Der erste Zugriff liest den Wert des ersten Eintrags (Eintrag Nummer Null!) im Feld mh, die Unterfeldnummer ist Null. Der zweite Aufruf von FDacc() schreibt diesen Wert in den zweiten Eintrag. Mit dem dritten Zugriff wird der dritte Eintrag inkrementiert.
Beispiel
Die meisten Funktionen der Datenbankmaschine benötigen Parameter 5 oder 6 nicht. Nicht benötigte Parameter werden von der DBE niemals gelesen und können daher problemlos weggelassen werden. Das spart ein oder zwei überflüssige mov-Anweisungen ein.
