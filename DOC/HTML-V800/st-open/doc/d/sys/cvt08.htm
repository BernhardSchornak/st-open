Ausgabe
Die Umwandlung mittels D2dec() und Q2dec() beinhaltet immer eine ordentliche Formatierung der Ausgabezeichenkette.
Die Formatanweisung wird im dritten Parameter (R08) übergeben:
Bit 31...24
Format
00 = 9 999.999
2C = 9,999.999
2E = 9.999,999
Bit 23...16
Fliesskomma
00 = keins
1...20 = Stellen
Bit 15...08
Vorzeichen
00 = ohne
?? = mit
Bit 07...00
Stellen
00 = 20
1...20
Übergeben Sie Null im dritten Parameter, wird eine vorzeichenlose zwanzigstellige Zahl zwischen 0 (mit 25 führenden Leerzeichen) und 18 446 744 073 709 551 615 ausgegeben. Die Funktion erzeugt im Zusammenspiel mit Festbreitenzeichensätzen perfekt formatierte Tabellen.
Formatierung
D2dec() wandelt [zahl] in eine in Dreiergruppen formatierte Dezimalzeichenkette ab Adresse [ziel] um. Die Dezimalstellenanzahl wird in Byte 00 des Parameters [stellen] festgelegt. Führende Nullen werden bei der Umwandlung automatisch in Leerzeichen umgewandelt, so dass Kolonnen mit Zahlen unterschiedlicher Grösse bei Verwendung von Festbreitenschriftarten korrekt dargestellt werden.
Byte 02 dieses Parameters legt die Anzahl der Pseudofliesskommastellen fest. Ist die Dezimalstellenanzahl Null, wird sie von D2dec() automatisch auf zehn gesetzt.
ACHTUNG: Der Ausgabepuffer muss mindestens 32 Byte gross sein!
Beschreibung
{
 DB s[32];
 D2dec(0x12345678, s, 516);    // 0x00000204
}
Die Zeichenkette s enthält nun "98.96". Der wirkliche Wert wäre eigentlich "3 054 198.96", die höchstwertigen Stellen werden durch die Beschränkung auf 4 Stellen jedoch abgeschnitten. Sie sollten daher auf den Wertebereich achten oder die Formatierung der Einfachheit halber generell auf 10 Stellen setzen (führende Nullen werden durch Leerzeichen ersetzt).
