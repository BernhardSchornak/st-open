Ausgabe
Die Umwandlung mittels D2dec() und Q2dec() beinhaltet immer eine ordentliche Formatierung der Ausgabezeichenkette. Verwenden Sie bitte d2dec(), um unleserlichen Zahlensalat in variablen Schriftbreiten auf den Bildschirm zu würgen!
Die Formatanweisung wird im dritten Parameter (R08) übergeben:
Bit 31...24
Format
00 = 9 999.999
2C = 9,999.999
2E = 9.999,999
Bit 23...16
Vorzeichen
00 = ohne
?? = mit
Bit 15...08
Fliesskomma
00 = keins
1...20 = Stellen
Bit 07...00
Stellen
00 = 20
1...20
Übergeben Sie Null im dritten Parameter, wird eine vorzeichenlose zwanzigstellige Zahl zwischen 0 (mit 25 führenden Leerzeichen) und 18 446 744 073 709 551 615 ausgegeben. Die Funktion erzeugt im Zusammenspiel mit Festbreiten-Zeichensätzen perfekt formatierte Tabellen.
Formatierung
Q2dec() wandelt [qwort] in eine Zeichenkette ab Adresse [ziel] um. Der dritte Parameter legt fest, ob und wie die Zeichenkette formatiert werden soll. Die Pseudofliesskommastellenanzahl darf nicht grösser als 20 sein!
ACHTUNG: Der Ausgabepuffer muss mindestens 32 Byte gross sein!
Beschreibung
{
 DQ q = 0x1234567890123;
 DB s[32];
 Q2dec(&q, s, 771817984);    // 0x2E010200
}
Die Zeichenkette s enthält nun "        3.202.559.734.582,11" (8 führende Leerzeichen).
