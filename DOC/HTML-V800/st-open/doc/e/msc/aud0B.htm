s08up() converts stereophonic 8 bit samples to double sampling rate. Only conversions 11.025 to 22.050 samples/s or 22.050 to 44.100 samples/s are supported. All other sampling rates require very complex computations beyond my math knowledge... :)
The function adds two consecutive samples and divides their sum by two. This interpolated value is stored between both samples to "stretch" the wave file to double sampling rate. The resulting [target] is twice as large as [source].
[source] and [target] must point to the first sample to convert. Hence, you may convert larger WAV files in chunks to reduce memory consumption.
Description
DD MyFunction(VOID *source, VOID *target)
{
 DD rc;
 rc = s08up(source, target, 0x2000);
 return(rc);
}
8,192 8-Bit samples from [source] are stretched to 16,384 8-Bit samples and stored in [target].
