Descriptions

0000

0000[MemEA] always contains a dword with a content of zero. All strings initally are set to offset zero - the address of an empty string. Some system functions evaluate this dword to validate the integrity of datafields. If a non-zero is found at 0000[MemEA], the content of the memory block is rejected and an ERR_NOT_A_FIELD message is returned.

0004

0004[MemEA] through 001F holds an optional field name. If no name is given, DatTools insert the number of the datafield in hexadecimal notation by default. This name is treated as a string, so it must be terminated with a trailing zero. Therefore, its maximum size is 27 byte. DatTools automatically truncate names exceeding this limit.

0020

0020[MemEA] holds the maximum size of any subfield in this datafield. It is calculated with the formula length * amount. This multiplication is done when the field is created, saving repeated calculations in functions where this value is required.

0024

0024[MemEA] holds the amount of datatypes stored in this datafield. Storing the count omits scans and superfluous calculations.

0028

0028[MemEA] holds control data for SecSys. If you don't use SecSys, the dword should be set to zero.

002C

002C[MemEA] holds the field number. It is used by the Loader for validation.

0030

0030[MemEA] holds the status word of the datafield.

0032

0032[MemEA] holds the required read permission for SecSys controlled datafields.

0033

0033[MemEA] holds the required write permission for SecSys controlled datafields.

0034

0034[MemEA] holds the current size of the datafield. This also is the offset where the next string will be written to.

0038

0038[MemEA] holds the maximum size of any entry in this datafield. For type 06 (FIX), it is the fixed size of each block in the subfield.

003C

003C[MemEA] holds the total amount of entries in each subfield.

0040

0040[MemEA] holds the type table of this datafield. The first not defined and all following entries in the table must be set to zero (no type).

0060

0060[MemEA] holds the offset table of this datafield. Unused offsets must be set to zero.

00E0

00E0[MemEA] up to 00FF[MemEA] are reserved for future expansions. All 32 bytes in this area should be set to zero.
