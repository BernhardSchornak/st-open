LD

   typedef struct _LD{
                      VOID *MemEA;
                      DD    MemSz;
                      DD    FldNr;
                      DD    LdCtl;
                      DD    MMoff;
                      VOID *FileN;
                      DD    Moffs;
                      DD    Foffs;
                      DD    AmtBy;
                      VOID *FileH;
                     } LD;
   typedef LD *PLD;
Definition
address of the allocated memory block
MemEA
optional: requested memory size (n * 4096 byte)
MemSz
optional: field number
FldNr
control and status flags
LdCtl
MemHandle (this is an address in the Loader table!)
MMoff
optional: filename
FileN
optional: memory offset (relative to MemEA)
Moffs
optional: file offset (relative to the 1st byte in the file)
Foffs
optional: amount of bytes to copy (0 = entire file)
AmtBy
optional: address of an FH structure
FileH
All unused optional parameters must be set to zero!
The LD structure only is required for the call to LDreq(), you may delete it after the call. To access the memory block, you either need the MemHandle (ld.MMoff) or the address of the memory block (ld.MemEA). You have to store the MemHandle. If you don't save it, you cannot access any Loader function - including LDfre()! It is recommended to store the MemHandle and the address (if required) in a global variable if you want to access a memory block outside the function where it was allocated.
Description
If you want to use the Loader, it is recommended to read the Loader Tutorial.
