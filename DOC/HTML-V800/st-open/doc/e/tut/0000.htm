Data Types

The naming of ST-Open's data types is similar to the naming in most assemblers. It saves a lot of typing work and defines the type explicitely. The subdivision into signed and unsigned only was defined to keep the compiler happy. Actually, most library functions treat data as quantities with a size of 8, 16, 32, 64 or 128 bit. In other words: They neither care about names nor signs and handle only one special data size. Whether you call a byte char, uchar or Hugo does not matter if you pass it to a function working with doublewords...
As a matter of fact: There are no data type definitions in assembler code at all - you either use a full register or partial registers like AX or CL. The same applies to addresses, they're always treated as 32 bit data. There's a short tutorial about data types and logic to learn more about data types.
