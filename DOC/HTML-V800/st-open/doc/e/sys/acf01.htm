FDvpt() is an additional definition for FDacc() to avoid casting orgies. Any qword returned by FDacc() is called 'qword' while qwords returned by FDvpt() are called 'pointer to void'. On error, FDvpt() returns the address of a system variable containing zero (empty string).
WARNING: Do not write strings to the returned address. Writing strings to a field directly might corrupt its content, leading to unpredictable errors and inconsitencies in the field's offset table. Use FDA_WRITE to write strings to your fields. This does not apply to numeric data. If you choose the proper size, you can write numeric data directly. That's faster than calling FDacc() repeatedly.
Description
{
 VOID *p;
 p = FDvpt(MH_STR, STR_MESSAGE, 0, FDA_ADDRESS, 0, 0);
}
A dword containing the address of STR_MESSAGE is returned by FDvpt() and stored in variable p.
