LDchk
Function
rc = LDchk();
Call
-
-

Input
DD
rc

Output
LDchk() periodically queries the time stamp of all allocated memory blocks. If the time stamp of a memory block expired, it is freed automatically. The timer interval is defined in the global variable CHK_DELAY (OS/2 HMSh format), the maximum time of inactivity is defined in the global variable CHK_REMOVE (OS/2 HMSh format). LDchk() subtracts the time stamp of each block from the current time and calls LDfre(), if the difference is a negative number. To prevent memory blocks from being removed by LDchk(), you should set the LDR_STATIC flag before you call LDreq(). Data fields and fields allocated with the LDR_AUTOSAVE option are stored automatically, all other memory blocks are removed without saving them.
Description
Dynamic memory management is an optional feature. You have to set CHK_DELAY and CHK_REMOVE to non-zero values, create and start a timer and provide a WM_TIMER procedure calling LDchk() to make use of it. Please read the files V700.c und hvar.h found in the V700 skeleton to get an overwiew how to implement the dynamic memory management feature properly.
