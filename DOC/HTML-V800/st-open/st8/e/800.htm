Version 8.0.0. (64 bit Windows)

This software is optimised for AMD-64 processors!
ST-Open's libraries provide about 300 functions developed for and tested with every day GUI applications. Even if all functions are coded in pure assembler, they can be used as dense and fast replacements for standard library functions of all High Level Languages following the 64 bit WinAPI calling conventions. The core library includes a high speed database engine, capable to handle 4,294,967,295 datafields with a size of 4 GB, each. Each datafield can hold up to 32 subfields with one of the defined datatypes: Byte, words, dwords, qwords, owords, 16*n byte blocks and dynamic strings. ST-Open's loader keeps loading of datafields transparent to the programmer, reducing required transactions to single calls: LDreq() to request a datafield, file or memory block, LDchg() to change an allocated block's size and LDfre() to free a block, datafield or file. Datafields are automatically saved if their content changed (can be skipped with a flag), files must be saved manually before the associated memory is freed (can be automated).
ST-Open's database engine (DBE) and memory management (Loader) are core components of the V800 system library. About 270 other functions for data conversion, string manipulation, time and date calculation and conversion, file system manipulation, API sandboxes (all used registers restored) and then some are included in the core library at the moment. Additional libraries provide ST-Open's filelist, ST-Open's OnLine-Help (OLH), the loader monitor, a register monitor with dump utilities for debugging, ST-Open's SecSys (security system) and an audio library.
ST-Open's V800 libraries are multi-lingual. You can switch between sixteen languages in the running program. Translations are stored in datafields, so every program can support as many languages as required by simply exchanging datafields on the fly.

OnLine Help

OnLine Help (OLH) is multi-lingual now and supports up to 16 languages. There are dialogs with two MLEs to translate texts comfortably by just typing them in. As an alternative, you may enter help texts and headlines in the OLH dialog directly.
